Issue 4: Incorrect lagna.  
Before:

num = -math.cos(ramc)

After:

num = -math.cos(ramc)

# atan2(num, den) yields the descending node; +180° gives the ascending node (Lagna)

The single change: % 360 → + 180) % 360.

Why it was wrong: atan2(num, den) computes the ecliptic point on the western horizon (the descendant — where signs are setting). The ascendant (Lagna) is always exactly 180° opposite — the point on the eastern horizon where signs are rising. 
Adding 180° before the modulo flips it to the correct rising point.

There's an important architecture point to explain first before giving you the steps.

The split you need to make
This app has two parts with different hosting requirements:

Part	Tech	Best host
Frontend	Next.js	Vercel (native support)
Backend	FastAPI + Chroma + LangGraph	Railway (persistent filesystem, Python runtime)
Vercel can run Python serverless functions, but your backend needs persistent disk (Chroma vector store rebuilds on startup, chat logs write to CSV). Vercel's serverless environment is stateless and resets between requests — it would break those features. Railway gives you a proper always-on Python process.

Part 1 — Deploy the backend to Railway
Go to railway.app → sign in with GitHub
New Project → Deploy from GitHub repo → select your repo
When asked for the root directory, set it to backend
Railway auto-detects Python. Make sure you have a requirements.txt in backend/ (run pip freeze > requirements.txt in your .venv first if you haven't)
Add environment variables in the Railway dashboard:

OPENAI_API_KEY=your-key
LANGCHAIN_API_KEY=your-key
Railway gives you a public URL like https://yourapp.up.railway.app — copy it
One thing to check: your backend/ needs a startup command. Railway will look for a Procfile or railway.json. Create this in backend/:


# backend/Procfile
web: uvicorn app.main:app --host 0.0.0.0 --port $PORT
